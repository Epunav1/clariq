# Clariq Technical Onboarding Guide

## 🎯 What is Clariq?

**Clariq** is an AI-powered analytics platform that lets e-commerce sellers ask their store data in plain English instead of using complex dashboards or SQL queries.

**Core Value Proposition:**
- Connect any e-commerce platform (Shopify, Amazon, TikTok, Alibaba, Jumia, WooCommerce, Etsy)
- Ask natural language questions: "Which products drive the most revenue?" 
- Get instant AI-powered answers with recommendations
- 99.99% uptime, enterprise security (SOC 2, 256-bit encryption)

**Target Users:** E-commerce sellers doing $1K-$10M+ annually across one or multiple platforms

## 📊 Product Architecture

### Frontend (Landing Page + Dashboard)
- **Technology:** HTML5 + Vanilla CSS3 + JavaScript (no frameworks)
- **Design System:** CSS variables for consistent theming
  - `--brand: #635bff` (purple)
  - `--ink: #0a2540` (navy) 
  - `--soft: #f6f9fc` (light background)

- **Pages:**
  - `index.html` - Multi-page landing site with:
    - Splash animation (tap to enter)
    - Intro animation (3 piano chord drops)
    - Hero section with CTA
    - Platform showcase (8 e-commerce platforms)
    - Trust/Security metrics
    - Pricing (3 tiers: Free/Starter, Pro $99/mo, Enterprise $249/mo)
    - Customer stories section
    - Dedicated pages: Product, Pricing, About, Support, Sign In, Sign Up
  - `index.html` also serves as the post-login dashboard after user authentication

### Backend API (FastAPI/Python)
- **Framework:** FastAPI >=0.100.0
- **Server:** Uvicorn (single worker, 0.0.0.0:PORT)
- **Language:** Python 3.9+

- **18+ Mounted Routers:**
  - `auth_routes_minimal.py` - JWT authentication, login, register
  - `query.py` - Natural language to SQL conversion & execution
  - `connections.py` - Platform connection management
  - `upload.py` - Data file uploads
  - `billing.py` - Stripe billing integration (optional)
  - Plus optional integrations: Anthropic AI, Snowflake, Shopify sync

- **Key Features:**
  - CORS middleware configured for `tryclariq.com` domains
  - Health check endpoint: `GET /api/health` → `{"api":"healthy","snowflake":"connected"}`
  - All optional integrations wrapped with try/except (graceful degradation)
  - Lifespan context manager with APScheduler for background jobs

### Database Layer
- **Primary:** SQLite at `backend/data/clariq.db` (persistent on Render filesystem)
- **Optional:** Snowflake (fully optional with `SNOWFLAKE_AVAILABLE` flag)
- **Strategy:** Graceful degradation - app works without Snowflake, returns 503 or error messages

### Background Jobs (APScheduler)
- **Config:** IntervalTrigger (not CronTrigger - cron can't handle >59 minute intervals)
- **Jobs:**
  - Shopify sync: every 60 minutes
  - Amazon sync: every 120 minutes
  - Report sending: Tuesdays 9 AM UTC
  - Milestone checks: every 30 minutes

## 🚀 Deployment Architecture

### Render.com (Current Platform)
Two services running in production:

1. **clariq-api** (Web Service)
   - GitHub branch: `main`
   - Build command: `pip install -r backend/requirements.txt`
   - Start command: `cd backend && python -m uvicorn main:app --host 0.0.0.0 --port $PORT --workers 1`
   - Status: ✅ LIVE at `https://clariq-api.onrender.com` and `https://api.tryclariq.com`

2. **clariq-web** (Static Site)
   - GitHub branch: `main`
   - Publish directory: `.` (root)
   - Serves: `index.html` and all static assets
   - Status: ✅ LIVE at `https://clariq-web.onrender.com` and `https://www.tryclariq.com`

### Domain Setup
- **Primary Domain:** `tryclariq.com` (purchased on Namecheap)
- **DNS Records (CNAME):**
  - `www` → `clariq-web.onrender.com` ✅
  - `api` → `clariq-api.onrender.com` ✅
  - `@` (root) → `clariq-web.onrender.com` ✅
- **SSL Certificates:** Auto-generated by Render for all domains

### Environment Variables (Required for API)
```
DATABASE_URL=sqlite:///data/clariq.db
JWT_SECRET=your-secret-key-here
ENVIRONMENT=production
PYTHONUNBUFFERED=1
ANTHROPIC_API_KEY=optional-for-ai-features
SNOWFLAKE_ACCOUNT=optional-for-snowflake
STRIPE_SECRET_KEY=optional-for-billing
```

## ⚙️ How to Get Started Locally

### 1. Clone Repository
```bash
git clone https://github.com/Epunav1/clariq.git
cd clariq
```

### 2. Backend Setup
```bash
cd backend

# Create virtual environment
python -m venv venv
source venv/bin/activate  # or `venv\Scripts\activate` on Windows

# Install dependencies
pip install -r requirements.txt

# Create data directory
mkdir -p data

# Set environment variables
export DATABASE_URL="sqlite:///data/clariq.db"
export JWT_SECRET="dev-secret-key-change-in-production"
export ENVIRONMENT="development"

# Run server
python -m uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

Server will be at `http://localhost:8000`
- Health check: `http://localhost:8000/api/health`
- API docs: `http://localhost:8000/docs` (Swagger UI)

### 3. Frontend Setup (Static)
```bash
cd ..

# Serve index.html locally
python -m http.server 3000
# or
npx http-server -p 3000
```

Frontend at `http://localhost:3000`

## 🔄 Development Workflow

### Making Backend Changes
1. Edit files in `backend/routes/` or `backend/ai/`
2. Server auto-reloads with `--reload` flag
3. Test via `http://localhost:8000/docs`
4. Commit to GitHub on `main` branch
5. Render auto-deploys within 1-2 minutes

### Making Frontend Changes
1. Edit `index.html`
2. Refresh browser
3. Test page navigation and interactions
4. Commit to GitHub on `main` branch
5. Render auto-deploys within 1-2 minutes

### Database Changes
- SQLite file persists at `backend/data/clariq.db`
- On Render, database persists on the filesystem (survives deployments)
- For schema changes, update SQLAlchemy models and run migrations if needed

## 📞 Key Contacts / Next Steps

**For New Developer:**
1. Clone the repo and follow "How to Get Started Locally"
2. Start with backend API (`backend/main.py`) to understand route structure
3. Review `backend/routes/auth.py` for authentication pattern
4. Test API endpoints via Swagger UI at `/docs`
5. Review `index.html` to understand frontend page system
6. Make a small change, commit, and watch Render deploy

**Common Issues:**
- Database not found? Make sure `backend/data/` directory exists
- Snowflake errors? They're caught and logged - app continues to work
- Port 8000 already in use? Change with `--port XXXX`
- JWT errors? Check `JWT_SECRET` environment variable is set
